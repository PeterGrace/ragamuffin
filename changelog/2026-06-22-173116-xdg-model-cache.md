# XDG-compliant model cache directory

**Date:** 2026-06-22 17:31:16

## Summary

`FastEmbedder` previously let `fastembed` use its default cache location, which
drops a `.fastembed_cache` directory into the current working directory. Running
`ragamuffin` from different places therefore re-downloaded the BGE-small ONNX
model and littered project trees with cache folders.

The model cache is now pinned to a stable, per-user location that follows the
XDG Base Directory specification.

## Changes

- **`Cargo.toml`**: added `directories = "6"` for OS-aware base-directory
  resolution.
- **`src/embedder/fastembed.rs`**:
  - New private `resolve_cache_dir()` helper resolving the cache path via
    `ProjectDirs::from("", "", "ragamuffin")`. On Linux this yields
    `$XDG_CACHE_HOME/ragamuffin` (default `~/.cache/ragamuffin`); macOS and
    Windows get their platform-appropriate equivalents. The directory is created
    with `create_dir_all` so `fastembed` can populate it.
  - `FastEmbedder::new()` now passes the resolved path to
    `TextInitOptions::with_cache_dir(...)`.
  - Added a Linux-only unit test asserting `XDG_CACHE_HOME` is honored and the
    `ragamuffin` subdirectory is created.
- **`src/error.rs`**: added `EmbedError::Cache(String)` for cache-directory
  resolution/creation failures (no `$HOME`, unwritable path, etc.).

## Rationale

The downloaded ONNX model is regenerable, non-essential data, so per the XDG
spec it belongs under `XDG_CACHE_HOME` (`~/.cache`) rather than
`XDG_CONFIG_HOME` (`~/.config`). Cache data there is safe to delete and is
conventionally excluded from dotfile backups.
