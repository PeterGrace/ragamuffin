## CLI Subcommands

| Subcommand | Description                                            |
|------------|--------------------------------------------------------|
| `add`      | Embed and store one already-compacted memory string    |
| `ingest`   | Chunk a text file and store each chunk with overlap    |
| `search`   | Embed query and return top-k hits as JSON              |
| `list`     | List all stored entries offline (no model load)        |
| `mcp`      | Launch MCP server over stdio (shim until Task 7)       |

Global flag `--store <DIR>` (default: `./ragstore`) applies to all subcommands.

## Design Notes

- `list` is fully offline: it calls `Store::open_existing()` which reads the dimension from the
  manifest, so no embedder is needed and no model is loaded.
- `search` and `list` emit JSON to stdout; `add` and `ingest` emit human-readable progress lines.
- `open_rag()` is a private helper that constructs `FastEmbedder` and `Rag`; it will work
  transparently once Task 6 provides the real implementation.
- The `unused import: clap::Parser` warning that existed in `main.rs` is now resolved because the
  real clap `Cli` struct (which requires `Parser` to be in scope) is imported from the library.
